# 自适应集成优化补充

在公平实验完成后，本轮只做了一个有边界的部署侧优化：Jar、Cellphone 使用 `wrist=4`，USB stick 使用 baseline；每个物体将 seed 0/1/2 的 best checkpoint 对同一观测产生的、尚未裁剪的 28 维确定性动作取均值。网络、数据和物理环境均未重新调参。

| 物体 | 路由 | 三模型集成 | normalized lift |
|---|---|---:|---:|
| Jar | wrist=4 | 7/20（35%） | 0.3583 |
| Cellphone | wrist=4 | 6/20（30%） | 0.3372 |
| USB stick | baseline | 3/20（15%） | 0.1506 |
| 合计 | 按物体路由 | **16/60（26.67%）** | **0.2820** |

![自适应集成对比](figures/adaptive_ensemble_comparison.png)

该结果高于公平实验中“任取一个 wrist=4 seed”的平均部署水平 14/60（23.33%），并把 Jar seed 0 的 0/20 提升为集成后的 7/20，说明均值集成能降低灾难性 seed 风险。但它低于根据既有结果事后选择方法的理论均值 17/60（28.33%），没有证明平均性能上限得到提升。

单模型 CPU 策略推理实测平均 0.91 ms；三模型集成在三个物体上平均 2.46 ms，约慢 2.7 倍，仍低于 60 Hz 控制周期的 16.7 ms。Isaac Gym 物理仍使用 GPU PhysX，旧 PyTorch 策略张量仍因 RTX 5070 Ti 的 sm_120 与 PyTorch 1.12.1+cu113（最高 sm_86）不兼容而在 CPU 执行。

## 结论边界

这是使用同一固定 20 条轨迹、并在看到公平实验结果后选择路由的探索性复测，不能作为独立测试集上的公平优越性结论。因此，公平实验报告中的 baseline/wrist=4 结论保持不变；集成仅作为可选的稳定部署方案。下一项高价值实验应固定该路由，在独立的初始位姿、摩擦系数和质量扰动 holdout 上验证，禁止继续依据当前 20 条轨迹选模型。
